Module 9: Denial of Service (DoS)

A Denial of Service (DoS) attack aims to disrupt the normal traffic of a targeted server, service, or network by overwhelming it with a flood of Internet traffic. When a server spends all its CPU and memory trying to answer fake requests from an attacker, legitimate users cannot load the website.

Insecure (Vulnerable)

Exposing heavy computational endpoints to anyone.

# INSECURE EXAMPLE (for learning only)
# Expensive query exposed globally

@app.route('/api/search_all')
def search_all():
    # DANGER: This endpoint searches 10,000,000 records.
    # An attacker can write a script to hit this endpoint
    # 100 times a second. The database will freeze trying
    # to process it, crashing the entire website for everyone.
    
    results = Database.execute("SELECT * FROM heavy_logs")
    return jsonify(results)
Secure (Mitigated)

Using pagination, caching, and Web Application Firewalls (WAF).

# SECURE IMPLEMENTATION
# Using Pagination and Caching

@app.route('/api/search_all')
def search_all():
    page = request.args.get('page', 1, type=int)
    
    # 1. Force pagination (only fetch 50 records max)
    # 2. Implement a CDN or caching layer like Redis
    # 3. Use a WAF (like Cloudflare) to automatically block
    #    IP addresses making unnatural amounts of requests.
    
    results = Database.execute("SELECT * FROM heavy_logs LIMIT 50")
    return jsonify(results)
