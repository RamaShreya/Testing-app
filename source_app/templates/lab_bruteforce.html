Module 6: Brute Force Attack

A Brute Force Attack is a method where an attacker submits many passwords or passphrases with the hope of eventually guessing correctly. The attacker systematically checks all possible passwords until the correct one is found. If an application lacks rate limiting, an attacker can make thousands of guesses per second.

Insecure (Vulnerable)

No limit on login attempts.

# INSECURE EXAMPLE (for learning only)
# Processing login without any rate limits

@app.route('/login', methods=['POST'])
def login():
    username = request.form.get('username')
    password = request.form.get('password')
    
    # DANGER: An attacker can send a script that hits this endpoint
    # 100,000 times trying different passwords for "admin".
    # The server will process every single attempt.
    
    user = User.query.filter_by(username=username).first()
    if user and check_password_hash(user.password_hash, password):
        return "Logged In"
    return "Failed"
Secure (Mitigated)

Implementing Rate Limiting and Account Lockout.

# SECURE IMPLEMENTATION
# Using Flask-Limiter or a custom tracking mechanism

# Restrict login attempts to 5 per minute per IP address
@limiter.limit("5 per minute")
@app.route('/login', methods=['POST'])
def login():
    username = request.form.get('username')
    password = request.form.get('password')
    
    # If an attacker tries 6 times, they will receive an HTTP 429
    # Too Many Requests error and the server will ignore them.
    
    user = User.query.filter_by(username=username).first()
    if user and check_password_hash(user.password_hash, password):
        return "Logged In"
    return "Failed"
