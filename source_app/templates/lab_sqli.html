Module 1: SQL Injection (SQLi)

SQL Injection happens when user input is directly concatenated into a database query. This allows an attacker to insert their own SQL commands.

Insecure (Vulnerable)

Using string formatting or concatenation.

# INSECURE EXAMPLE (Active in VULN_MODE)
username = request.form.get('username')

# DANGER: Input is directly concatenated
query = f"SELECT * FROM users WHERE username='{username}'"
db.session.execute(text(query))
Secure (Mitigated)

Using Parameterized Queries or ORMs.

# SECURE IMPLEMENTATION (Active in SAFE_MODE)
username = request.form.get('username')

# Safe: Using ORM filtering
user = User.query.filter_by(username=username).all()
