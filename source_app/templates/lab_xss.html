Module 2: Cross-Site Scripting (XSS)

XSS occurs when an application includes untrusted data in a web page without proper validation or escaping. If we allow users to leave product reviews, an attacker could inject malicious scripts into the page viewed by other users.

Insecure (Vulnerable)

Rendering raw, unescaped user input in HTML.

<!-- INSECURE EXAMPLE (for learning only) -->
<!-- The 'safe' filter disables auto-escaping -->

<div class="user-review">
    <h5>{% raw %}{{ review.author }}{% endraw %}</h5>
    
    <!-- If review content is "<script>stealCookies();</script>", -->
    <!-- the browser will execute the script! -->
    <p>{% raw %}{{ review.content | safe }}{% endraw %}</p>
</div>
Secure (Mitigated)

Using framework auto-escaping (Context-aware output encoding).

<!-- SECURE IMPLEMENTATION -->
<!-- Relying on default Jinja2 behavior -->

<div class="user-review">
    <h5>{% raw %}{{ review.author }}{% endraw %}</h5>
    
    <!-- Flask/Jinja2 automatically converts special characters -->
    <!-- like < and > into &lt; and &gt;. -->
    <!-- The script is safely displayed as raw text, not executed. -->
    <p>{% raw %}{{ review.content }}{% endraw %}</p>
</div>
