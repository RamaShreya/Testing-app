Module 12: Business Logic Flaw

Business logic flaws are vulnerabilities that arise from faulty application design and implementation. They allow attackers to manipulate the intended behavior of the application (e.g., getting items for free or skipping payment steps).

Mock Checkout Test

Try to manipulate the checkout process. Can you get the price to be negative?

Item: Premium Subscription

Base Price: $100.00

Available: SAVE10 (10%), WELCOME50 (50%)

SUBTOTAL (x{{ quantity }})

${{ "%.2f"|format(total * quantity) }}

DISCOUNT

-${{ "%.2f"|format(discount) }}

FINAL PRICE

${{ "%.2f"|format(final_price) }}

Exploit Scenario

Toggle **Vulnerability Mode: ON** and enter a negative quantity (e.g. -1). In many broken apps, this will cause the total to become negative, effectively "charging" the store!

Insecure (Vulnerable) 
# INSECURE (Active in VULN_MODE)
qty = int(request.form.get('quantity'))
# DANGER: Trusting user input
final_price = (price * qty) - discount
Secure (Mitigated) 
# SECURE (Active in SAFE_MODE)
qty = int(request.form.get('quantity'))
if qty < 1:
    qty = 1
final_price = (price * qty) - discount
