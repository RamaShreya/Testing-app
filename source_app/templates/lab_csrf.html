Module 5: Cross-Site Request Forgery (CSRF)

CSRF is an attack that forces an authenticated user to execute unwanted actions on a web application in which they're currently authenticated. If forms are unprotected, an attacker can embed malicious requests into third-party sites that the victim's browser will execute unknowingly.

Insecure (Vulnerable)

Standard HTML form with no hidden verification tokens.

<!-- INSECURE EXAMPLE (for learning only) -->
<!-- No CSRF token present -->

<form action="/update_email" method="POST">
    <label>New Email Address:</label>
    <input type="email" name="new_email">
    
    <!-- An attacker can host this exact form on their -->
    <!-- own malicious website and use Javascript to -->
    <!-- automatically submit it when you visit! -->
    
    <button type="submit">Update</button>
</form>
Secure (Mitigated)

Requiring a cryptographic token matching the user session.

<!-- SECURE IMPLEMENTATION -->
<!-- Utilizing Anti-CSRF Tokens (e.g., via Flask-WTF) -->

<form action="/update_email" method="POST">
    <!-- This hidden token is securely generated by the -->
    <!-- server and unique to the user's current session. -->
    <!-- The attacker cannot guess it! -->
    {% raw %}{{ form.csrf_token }}{% endraw %}
    
    <label>New Email Address:</label>
    <input type="email" name="new_email">
    
    <button type="submit">Update</button>
</form>
