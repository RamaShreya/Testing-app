Module 4: Insecure Direct Object Reference (IDOR)

IDOR occurs when an application provides direct access to objects based on user-supplied input without verifying if the user has permission to access that specific object.

Insecure (Vulnerable)

Fetching data strictly based on the URL parameter.

# INSECURE (Active in VULN_MODE)
@app.route('/orders/<user_id>')
def view_orders(user_id):
    # DANGER: No ownership check!
    orders = Order.query.filter_by(user_id=user_id).all()
    return render_template('orders.html', orders=orders)
Secure (Mitigated)

Enforcing authorization checks against the current session.

# SECURE (Active in SAFE_MODE)
@app.route('/orders/<user_id>')
def view_orders(user_id):
    # Check if ID matches logged-in user
    if current_user.id != user_id:
        return "Unauthorized", 403
        
    orders = Order.query.filter_by(user_id=user_id).all()
