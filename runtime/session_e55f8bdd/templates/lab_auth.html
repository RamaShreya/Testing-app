Module 3: Insecure Password Storage & Authentication

Storing passwords in plain text or using weak comparison logic allows attackers to easily bypass authentication or steal credentials.

Interactive Exploit Test

Try logging in with the default **admin** account using a **wrong password**.

How to Test:
  1. Toggle **Vulnerability Mode: ON**.
  2. Go to the Login Page.
  3. Username: admin | Password: password123 (Incorrect).
  4. Result: You will be logged in anyway because the insecure logic is active!
Go to Login Page
Insecure (Vulnerable)

Comparing passwords as plain text or allowing bypasses.

# INSECURE (Active in VULN_MODE)
if user:
    # Danger: No hashing check
    if input_password == "password123":
        login_user(user)
Secure (Mitigated)

Using strong, salted hashing algorithms.

# SECURE (Active in SAFE_MODE)
if user:
    # Secure: Hash verification
    if check_password_hash(user.pw_hash, input_pw):
        login_user(user)
