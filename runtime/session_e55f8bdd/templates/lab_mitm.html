Module 10: Man-in-the-Middle (MITM)

A Man-in-the-Middle (MITM) attack occurs when an attacker secretly intercepts and possibly alters the communication between two parties who believe they are directly communicating with each other. For example, if you connect to a malicious public Wi-Fi network, the attacker can view everything you send.

Insecure (Vulnerable)

Sending sensitive data over unencrypted HTTP.

<!-- INSECURE EXAMPLE (for learning only) -->
<!-- Form submitting over standard HTTP -->

URL: http://www.testingapp.com/checkout

<form action="http://www.testingapp.com/pay" method="POST">
    <input type="text" name="credit_card_number">
    <button type="submit">Pay Now</button>
</form>

<!-- Because the URL is HTTP, the credit card is sent -->
<!-- across the internet in plain text. Any router -->
<!-- between the user and the server can read it! -->
Secure (Mitigated)

Enforcing HTTPS with valid SSL/TLS certificates.

<!-- SECURE IMPLEMENTATION -->
<!-- Form submitting over encrypted HTTPS -->

URL: https://www.testingapp.com/checkout

<form action="https://www.testingapp.com/pay" method="POST">
    <input type="text" name="credit_card_number">
    <button type="submit">Pay Now</button>
</form>

<!-- Because the URL is HTTPS, the data is scrambled -->
<!-- using mathematically secure encryption before it -->
<!-- leaves the browser. An attacker only sees gibberish. -->
