Module 8: Ransomware

Ransomware is malicious software that encrypts a victim's files, rendering them inaccessible. The attacker then demands a ransom to restore access. If an application allows unfiltered file uploads or runs with excessive system privileges, ransomware can infect the server.

Insecure (Vulnerable)

Allowing executable file uploads and running as Root.

# INSECURE EXAMPLE (for learning only)
# Unrestricted File Upload

@app.route('/upload_profile_pic', methods=['POST'])
def upload():
    file = request.files['image']
    
    # DANGER: The app blindly saves any file. 
    # An attacker can upload a malicious script (e.g. ransomware.py).
    # If the app runs with Admin/Root system privileges,
    # the script can encrypt the entire server!
    
    file.save(os.path.join('/uploads', file.filename))
    return "Uploaded!"
Secure (Mitigated)

Strict file validation and offline backups.

# SECURE IMPLEMENTATION
# Strict validation and least-privilege

ALLOWED_EXTENSIONS = {'png', 'jpg', 'jpeg'}

def allowed_file(filename):
    return '.' in filename and \
           filename.rsplit('.', 1)[1].lower() in ALLOWED_EXTENSIONS

@app.route('/upload_profile_pic', methods=['POST'])
def upload():
    file = request.files['image']
    
    # 1. Enforce strict extension checks
    # 2. Run the application as a restricted user (no root access)
    # 3. Maintain regular, offline database backups!
    
    if file and allowed_file(file.filename):
        safe_name = secure_filename(file.filename)
        file.save(os.path.join('/uploads', safe_name))
        return "Uploaded securely!"
    return "Invalid file type."
