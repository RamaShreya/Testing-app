Module 17: CSRF Attack Simulation

This page simulates a **Malicious Website** trying to exploit the CSRF vulnerability in this application. It will attempt to force your browser to place an order without your consent.

How it works: In a real attack, this form would be hidden (CSS display: none) and submitted automatically via Javascript when you visit the attacker's site.
Malicious Site: "Get Free Rewards!"

You just won a prize! Click the button to claim it (this is a trap).

MOCK MALICIOUS FORM

Clicking this button submits a POST request to /checkout. Since there is no CSRF token, it will succeed!

Vulnerable Code

No CSRF token present in the real checkout form.

<form action="/checkout" method="POST">
  <!-- Missing CSRF Token! -->
  <button>Place Order</button>
</form>
Secure Code

Using Flask-WTF or CSRFProtect.

<form action="/checkout" method="POST">
  {% raw %}{{ form.csrf_token }}{% endraw %}
  <button>Place Order</button>
</form>
