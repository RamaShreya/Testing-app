Module 9: Unsafe File Upload

Allowing users to upload files without validating the file type, content, or extension can allow an attacker to upload malicious scripts (like a web shell) that can execute code on the server.

Upload File Test

Try uploading a safe image (PNG/JPG). Then, try to upload a restricted file type like .html or .php.

If Vulnerability Mode is ON, any file extension will be accepted!

Exploit Scenario

Upload a file named shell.html containing <script>alert('XSS via File Upload')</script>. Then navigate to /static/lab_uploads/shell.html to trigger it.

Insecure (Vulnerable) 
# INSECURE (Active in VULN_MODE)
file = request.files.get('file')
# No extension or type check!
file.save(os.path.join(path, file.filename))
Secure (Mitigated) 
# SECURE (Active in SAFE_MODE)
allowed = {'png', 'jpg', 'jpeg', 'gif'}
if ext not in allowed:
    return "Invalid type"
file.save(os.path.join(path, safe_name))
